# Research: Claude / Agent Skills — Insights for the Book

Research the following topics and provide insights to be used in the book.

---

## Topics to Cover

### 1. Invest in Building Skills and Workflows of Skills
- What are Agent Skills and how do they work?
- Why skills are model-agnostic — the same skill can run on Claude, GPT, Gemini, or any future model
- How the skills format (e.g. SKILL.md + scripts) avoids vendor lock-in and tool lock-in
- Building reusable workflows composed of multiple skills chained together
- Why investing in skills is more durable than investing in any specific agent platform

### 2. Train Your People to Become Agentic Employees
- What does it mean to be an "agentic employee"?
- The best employees will create their own skills and share them with their teams
- How skill-sharing within organizations creates compounding value — the network effect of expertise
- What skills and mindsets separate high-performing agentic employees from the rest
- How to run internal skill-building programs and communities of practice

### 3. Enterprise-Scale Skills Management
- What tools exist today to manage, version, discover, and govern skills at enterprise scale?
- Current state: tools are getting more stable but still maturing
- What significant progress is expected in 2026 — governance, discoverability, security, observability
- How to think about a skills catalog or skills library as a strategic enterprise asset

### 4. Unlimited Tokens as the Best Perk to Attract and Retain Talent
- Why offering unlimited AI token access is becoming a top-tier employee benefit
- The competitive signal it sends: we trust you to use the best tools available
- How to structure token access policies — per-seat, pooled, team-based
- The ROI argument: a few thousand dollars in tokens per person can unlock 10x productivity
- Compare to other perks — why tokens beat gym memberships and free lunches

### 5. Create Space for Employees to Experiment
- Offer a monthly budget (e.g. $50/person) for employees to subscribe to AI tools of their choice
- How to set security guardrails while still enabling genuine experimentation
- The importance of psychological safety — experimentation must be celebrated, not just tolerated
- How bottom-up experimentation surfaces the best skills and workflows for the whole organization
- Examples of how small experiment budgets have produced outsized organizational learning

### 6. Creating Skills Will Replace Building Software and Traditional Architecture
- How the role of software engineers and architects shifts in a world where skills replace bespoke code
- Skills as the new "apps" — the analogy: model = processor, agent runtime = OS, skills = software
- What happens to custom software development spend when a skilled agent + a SKILL.md can do the job?
- The long-term decline of one-off internal tooling and the rise of composable, agent-executable skills
- What architects and senior engineers should be building toward in this transition

### 7. Impact on IT Architecture and Software Spending Long-Term
- How enterprise software budgets will migrate from licenses and SaaS subscriptions to token usage
- Example: a $400k token budget replacing $400k in legacy software licenses
- What this means for IT procurement, vendor relationships, and enterprise architecture decisions
- Which categories of software are most vulnerable to displacement by agent skills (workflow tools, RPA, BI, etc.)
- How to model and forecast the shift from capex/opex software spend to consumption-based token spend

### 8. Lead from the Top — Executive Accountability for AI Adoption
- Why AI transformation fails without visible, committed leadership from the top
- Accenture as a case study: including AI/token usage as a condition for promotions within the company
- How to tie AI adoption and skill-building to performance reviews, career ladders, and incentive structures
- The message it sends when executives use AI tools themselves — modeling the behavior they want to see
- How to create a "lead from the top" AI adoption playbook for the C-suite
